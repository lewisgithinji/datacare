## RLS Policy Logic

```sql
-- Example: whatsapp_conversations RLS policy

CREATE POLICY "Users can view conversations in their organizations"
  ON whatsapp_conversations FOR SELECT
  TO authenticated
  USING (
    organization_id IN (
      -- This subquery is the key!
      SELECT organization_id
      FROM whatsapp_team_members
      WHERE user_id = auth.uid()  -- Current user's ID from JWT
        AND is_active = true
    )
  );
```

**How it works:**
1. User makes request with JWT token
2. Supabase extracts `user_id` from token â†’ becomes `auth.uid()`
3. RLS policy runs the USING clause
4. Subquery finds user's organization(s) from `whatsapp_team_members`
5. Only returns conversations where `organization_id` matches

**Result:**
- âœ… User A (org 1) sees only org 1 conversations
- âœ… User B (org 2) sees only org 2 conversations
- âœ… User C (org 1 + org 2) sees both organizations
- âŒ User D (not linked) sees nothing
